APIs based on HTTP

The most commonly used API based on HTTP is the Fetch API, which can be used to make HTTP requests from JavaScript. The Fetch API replaces the XMLHttpRequest API.

  1. Fetch API

    The fetch function is primarily a JavaScript function used in web browsers to make HTTP requests. It is part of the Fetch API, which provides a modern, promise-based way to perform network requests1. However, as of recent updates, the Fetch API has also been stabilized and included in Node.js, allowing developers to use fetch in server-side JavaScript environments as well2. This means you can use fetch both in the browser and in Node.js applications without needing additional libraries.

  2. XMLHttpRequest

    The XMLHttpRequest object is a built-in browser object in JavaScript, primarily used for making HTTP requests in web browsers12. It allows you to fetch data from a server without refreshing the page, making it a key component in AJAX (Asynchronous JavaScript and XML) programming. While XMLHttpRequest is not natively available in Node.js, you can use libraries like xmlhttprequest or xhr2 to achieve similar functionality in a Node.js environment3.
