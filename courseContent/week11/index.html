Intro to WebPack & Asset Bundling

Webpack is a popular open-source JavaScript module bundler. In simpler terms, it takes all the code from your application, transforms and bundles it together into one or more optimized static assets ready for deployment.

Installation


    mkdir webPack
    ...\ice\webPack> npm init
    ...\ice\webPack> npm install webpack webpack-cli webpack-dev-server html-webpack-plugin copy-webpack-plugin --save-dev

Basic Setup

Configuration

WebPack Config is needed to extend webPack functionality.

Concepts

  • Entry

    A rule to consider: one entry point per HTML page. SPA: one entry point, MPA: multiple entry points.

  • Output

    The top-level output key contains a set of options instructing webpack on how and where it should output your bundles, assets, and anything else you bundle or load with webpack.

    • output.filename
    • output.path
  • Module

    The module property in a Webpack configuration file (webpack.config.js) is used to define how different types of modules within your project should be treated. This includes specifying rules for loaders that process files, such as JavaScript, CSS, images, and more.

    • Modules

      Webpack modules can express their dependencies in a variety of ways.

      • ES2015 import statement
      • @import statement inside of a css/sass/less file.
      • An image url in a stylesheet url(...) or HTML <img src="..."> file.
    • Rule.test property is used to identify which files should be processed by a specific loader. It acts as a condition to match file paths against a regular expression. When a file path matches the regular expression specified in Rule.test, the corresponding loader(s) will be applied to that file.
    • Loader

      A Webpack loader is a module that transforms files as they are processed by Webpack. Loaders allow you to preprocess files before they are bundled, enabling you to bundle not just JavaScript, but also other static resources like CSS, images, and HTML. They are essential for transforming and bundling various types of files in your project.

  • Plugins

    The HtmlWebpackPlugin simplifies creation of HTML files to serve your webpack bundles. The plugin will generate an HTML5 file for you that includes all your webpack bundles in the body using script tags.

The following is the scripts in package.json to specify which config file to be used.

                
    "scripts": {
        "test": "echo \"Error: no test specified\" && exit 1",
        "start": "webpack serve --config config\\webpack.config.js --mode development",
        "build": "webpack --config config\\webpack.config.js --mode production"
